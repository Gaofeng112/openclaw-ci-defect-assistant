## 失败的尝试

- ❌ 早期尝试走 OpenAPI `/v3/task/create`，失败原因是租户权限问题，返回过 `Resource not belong to tenant`。当前正确方向是复用网页登录态调用 Web API `/api/v2/tasks`。
- ❌ `teambition_v3.har` 没有点击“完成创建”，所以没有真实创建接口，只能提字段和已有任务样本。
- ❌ 第一次真实提交失败，返回：
  - `FileTokenInvalidScope`
  - 原因：payload 里复用了 v4 抓包中的 `noteRtfValue.url` 富文本临时文件 token。
  - 处理：在 `app\tools\teambition_payload.py` 中移除 `noteRtfValue`、`noteRtfData`、`noteRenderMode`，只提交普通 `note`。
- ❌ 第二次重新创建时，用 PowerShell here-string 临时写中文 JSON，导致中文变成 `?`。后续应使用 UTF-8 文件或避免用 PowerShell 管道传中文。

## 关键决策

- 决策：保留旧 `app\tools\teambition_tool.py` 的 OpenAPI 创建逻辑，不直接替换。
  - 原因：旧逻辑仍是 `/v3/task/create`，且权限路径和现在验证通过的 Web API 不同。
- 决策：新增独立脚本做 `/api/v2/tasks` 真实提交。
  - 原因：先验证真实创建链路，避免直接影响 DingTalk 主流程。
- 决策：Cookie 只保存到 `runtime\teambition_har\teambition_headers.json`。
  - 原因：Cookie 是敏感凭证，`runtime/` 已被 Git 忽略，不能打印或提交。
- 决策：真实提交必须显式传 `--confirm`。
  - 原因：避免误创建公司项目缺陷。
