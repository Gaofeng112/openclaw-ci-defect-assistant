# 交付说明

## 给别人什么

直接把整个仓库打包成 zip，或者让对方从 git 拉取仓库。

需要包含：

```text
app/
scripts/
configs/
openclaw-plugin/
pyproject.toml
README.md
.env.example
```

不要包含：

```text
.env
.venv/
runtime/
openclaw-plugin/node_modules/
openclaw-plugin/dist/
*.har
*.log
```

## 对方怎么安装

前提：

- Windows
- PowerShell
- Python 3.11+
- Node.js / npm
- 已安装并配置 OpenClaw

在项目根目录执行：

```powershell
.\scripts\install.ps1
```

安装脚本会自动创建 `.venv/`、`runtime/` 和 `.env`。

如果只想检查 CLI 和插件，不安装到 OpenClaw：

```powershell
.\scripts\install.ps1 -SkipOpenClawInstall
```

安装完成后，检查：

```powershell
openclaw plugins inspect openclaw-ci-defect-assistant
openclaw plugins doctor
openclaw status --deep
```

## 对方需要改什么

安装后编辑 `.env`：

```powershell
notepad .env
```

至少填写：

```text
JENKINS_BASE_URL=
JENKINS_USER=
JENKINS_TOKEN=

TEAMBITION_APP_ID=
TEAMBITION_APP_SECRET=
TEAMBITION_ORG_ID=
TEAMBITION_OPERATOR_ID=
```

如果对方和你使用同一个 Teambition 项目、同一套缺陷表单，`configs/teambition.yaml` 和 `configs/teambition_bug_form.v1.yaml` 里的项目 ID、字段 ID 可以直接复用。

如果对方使用另一个 Teambition 项目，必须重新抓取或确认这些 ID：

```text
project_id
tasklist_id
stage_id
taskflowstatus_id
scenariofieldconfig_id
customfield_id
customfield option id
executor/member id
```

## 为什么不是一个文件就能用

这个项目不是纯插件。它由两层组成：

```text
OpenClaw plugin -> 本地 Python CLI -> Jenkins / Teambition
```

插件只负责让 OpenClaw 能调用工具。真正执行 Jenkins 和 Teambition 的逻辑在 Python CLI 里，所以必须同时交付仓库代码、配置文件和插件。

## Teambition ID 能不能一起给别人用

能，但只限同一个 Teambition 项目和同一套表单。

这些 ID 已经放在：

```text
configs/teambition.yaml
configs/teambition_bug_form.v1.yaml
```

不要把登录态、cookie、HAR、`.env` 一起给别人。这些是本机私有数据。
